# 功能一：短线预测统计
1. 目前在跟一B站博主的每日股票预测，他每天推荐5支短线股，我想观察一段时间他推荐的平均收益率。
2. 输入：我每天把股票代码导入，你帮我查询每天的开盘价和收盘价，计算每支的收益和平均收益（5支都买入一手）
3. 对股票代码分类，分析不同股票的行业趋势
4. 对不同维度的统计生成图表方便分析
5. 对源输入信息每支股票提取如下关键信息：日期，股票代码，公司名称，题材。保存到.csv中
6. 因为我每天会更新5支股票，所以每次摘取数据后做本地缓存，避免每次都要摘取历史数据。所以从上面的.csv中先检查本地是否有缓存，如果没有则从后台拉取股票开盘和收盘信息，再进行统计

# 功能三：理财管理（可开发版本）
目标：管理当前存款、基金、股票等投资资产，保证账务可追踪、余额可核对、收益可统计，并提供可配置的 AI 辅助分析能力。

## 3.1 范围定义
1. 本模块只记录存款和投资类（银行、支付宝、微信、基金、股票）相关资金流。
2. 日常消费流水暂不纳入（后续如有需要可新增“生活流水”模块）。
3. 核心原则：账务计算必须由业务代码和数据库事务保证准确性，AI 只做分析建议，不直接改账。

## 3.2 核心功能（MVP）
1. 资产总览与分布
   - 支持按资产类型查看当前资产分布：银行、支付宝、微信、基金、股票。
   - 银行账户支持多账户管理（如光大、中国、招商、农行、浙商、建行等）。
   - 基金需精确到基金标的，股票需精确到股票 ID。

2. 交易与转账
   - 支持记录收入、转账、基金买卖、股票买卖等交易。
   - 支持账户间转账，自动生成转账记录，并保证转出/转入余额一致。
   - 支持记录固定收入（如工资）和大额支出（如急用），并标注来源与类型。

3. 持仓与收益
   - 支持维护基金/股票持仓明细（买入、卖出、数量、成本）。
   - 支持统计储蓄、基金、股票收益（已实现收益与持仓收益分开展示）。
   - 基金/股票买卖历史可查询可追溯。

4. Web 面板
   - 提供资产总览、交易录入、交易历史、收益统计等基础界面。
   - 支持按时间范围筛选与查询。

5. 登录认证
   - MVP 先实现账号密码登录。
   - 后续可扩展 rsa.pub 相关认证方式。

## 3.3 技术选型建议
1. 数据库：统一使用 PostgreSQL（开发与生产环境保持一致，避免方言差异与迁移风险）。
2. 向量数据库：MVP 非必需。先不引入，避免系统复杂度过高。
3. RAG：可用于 AI 问答与解释（例如“为什么本月收益下降”），不用于账务计算。
4. 模型接入：采用可配置 Provider 设计，支持 openai、deepseek、gemini、openrouter、qwen 等。

## 3.4 数据模型（草案）
1. users：用户信息、密码哈希、状态。
2. accounts：账户信息（类型、名称、所属用户、当前余额）。
3. assets：资产标的信息（类型、标的代码、名称）。
4. transactions：交易流水（交易类型、金额、时间、来源账户、目标账户、备注）。
5. positions：持仓记录（资产标的、数量、成本、更新时间）。
6. income_expense_tags：收支类型（工资、转账、急用等，可扩展）。

## 3.5 目录结构建议
1. backend/：API、业务逻辑、数据库访问。
2. frontend/：Web 界面。
3. core/：领域模型与通用计算逻辑（收益计算、校验规则）。
4. data/：缓存与导入导出数据。
5. docs/：需求文档、接口文档、设计说明。

## 3.6 里程碑（建议）
1. M1（基础账务）
   - 完成账户、交易、转账、资产总览。
2. M2（投资与收益）
   - 完成基金/股票持仓、历史记录、收益统计。
3. M3（Web 与认证）
   - 完成 Web 面板和账号密码登录。
4. M4（AI 辅助）
   - 完成模型配置接入、财务状态分析和建议输出。
